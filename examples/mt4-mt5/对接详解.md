# MT4/MT5 对接详解

> 本文是 [README.md](../README.md) 的详细版,深入讲解 MT 与 C# 服务的对接流程,并附 2 个简单插件示例。

## 一、对接方式对比

| 方式 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **纯 MQL 实现** | 简单策略,独立运行 | 无外部依赖 | 算法库少,调试简陋 |
| **EA + C# 服务(ZeroMQ)** | 复杂算法,需要 ML/数据库 | NuGet 海量库,VS 调试 | 需要维护两套代码 |
| **C# Manager API 直连** | 企业级跟单/风控 | 多账户管理,无需 EA | 需经纪商授权 |

## 二、ZeroMQ 对接详细步骤

### 2.1 准备工作

**MT5 端安装 mql-zmq 库**:
1. 下载 https://github.com/dingmaotu/mql-zmq
2. 把以下文件复制到 MT5 数据目录:
   ```
   libzmq.dll    → MQL5/Libraries/
   Zmq.mqh       → MQL5/Include/Zmq/
   ```

**查看 MT5 数据目录**:
- MT5 → 文件 → 打开数据文件夹

### 2.2 通信模式选择

| 模式 | 特点 | 适用场景 |
|------|------|---------|
| **REQ-REP** | 严格请求-响应配对 | EA 每次请求等响应(本示例) |
| **PUB-SUB** | 服务端广播,客户端订阅 | 服务端主动推送信号(低延迟) |
| **ROUTER-DEALER** | 异步多客户端 | 高并发,多 EA 共享服务 |

**本示例用 REQ-REP 模式**:
```
EA (REQ)                  C# 服务 (REP)
  │                          │
  ├─ send(行情) ───────────> │
  │                          ├─ 处理
  │  <─────────── send(信号)─┤
  ├─ recv()                  │
```

### 2.3 协议设计

**请求(EA → C# 服务)**:
```json
{
  "symbol": "EURUSD",
  "price": 1.1050,
  "time": "2026-07-18T10:00:00"
}
```

**响应(C# 服务 → EA)**:
```json
{
  "ok": true,
  "msg": "已收到 EURUSD @ 1.10500",
  "server_time": "2026-07-18T10:00:01"
}
```

**设计原则**:
1. 用 JSON 而非二进制,便于调试(代价是性能略低)
2. 字段名 snake_case 与 MQL 风格一致
3. 用 `ok` 字段统一表示成功/失败,便于 EA 判断

### 2.4 调试流程

**步骤 1: 不依赖 MT 验证 C# 服务**

```bash
# 终端 1: 启动服务
dotnet run --project examples/MtBridge.Server

# 终端 2: 启动 C# 测试客户端
dotnet run --project examples/MtBridge.Client
```

期望输出:
```
[发送] {"symbol":"EURUSD","price":1.105,...}
[收到] {"ok":true,"msg":"已收到 EURUSD @ 1.10500",...}
```

**步骤 2: MT5 终端集成**

1. 启动 C# 服务:`dotnet run --project examples/MtBridge.Server`
2. 把 [SignalClientEa.md](SignalClientEa.md) 中的 MQL5 源码复制到 `MT5\MQL5\Experts\SignalClientEa.mq5`
3. 在 MetaEditor 中按 F7 编译
4. 在 MT5 终端:
   - 顶部"自动交易"按钮必须点亮(绿色)
   - 拖拽 EA 到 EURUSD 图表
   - 弹窗:勾选"允许实时自动交易" → 确定

**步骤 3: 验证**

查看 MT5 终端"专家"标签页,应该看到:
```
【初始化成功】已连接 tcp://127.0.0.1:5556
【发送】{"symbol":"EURUSD","price":1.10500,"time":"2026.07.18 10:00:00"}
【收到】{"ok":true,"msg":"已收到 EURUSD @ 1.10500",...}
```

C# 服务端控制台应该看到:
```
[收到] {"symbol":"EURUSD","price":1.10500,...}
[回复] {"ok":true,"msg":"已收到 EURUSD @ 1.10500",...}
```

### 2.5 常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| EA 不发送请求 | "自动交易"未点亮 | MT5 顶部按钮点绿 |
| EA 连接失败 | C# 服务未启动 | 先启动 `dotnet run --project examples/MtBridge.Server` |
| EA 收不到响应 | 防火墙拦截 5556 | 防火墙放行 5556 端口 |
| 编译报错找不到 Zmq.mqh | mql-zmq 未安装 | 复制到 `MQL5/Include/Zmq/` |
| 运行报错找不到 libzmq.dll | libzmq.dll 未安装 | 复制到 `MQL5/Libraries/` |
| C# 服务卡住不响应 | ZMQ_REQ 模式下 EA 未 send 直接 recv | 检查 EA 端 send/recv 配对 |

## 三、插件示例 1: Custom Indicator(自定义指标)

**Custom Indicator** 只能在图表上画线,**不能下单**。常用于辅助分析。

### 3.1 业务场景
画一条"价格偏离均线"的副图指标,直观显示超买超卖。

### 3.2 源码

```mql5
//+------------------------------------------------------------------+
//|                                       PriceDeviation.mq5         |
//|       华鑫融汇 - 自定义指标示例:价格偏离均线                       |
//+------------------------------------------------------------------+
// 【目的】画一条副图线,显示当前价偏离 MA20 的百分比
//        正值 = 价格在均线上方(超买倾向)
//        负值 = 价格在均线下方(超卖倾向)
//+------------------------------------------------------------------+

#property indicator_separate_window    // 显示在副图(非主图)
#property indicator_plots 1
#property indicator_label1 "Deviation"
#property indicator_type1   DRAW_LINE
#property indicator_color1   clrBlue
#property indicator_style1  STYLE_SOLID
#property indicator_width1  2

input int MaPeriod = 20;  // MA 周期

double DeviationBuffer[];  // 指标数据缓冲区
int    g_maHandle;

//+------------------------------------------------------------------+
//| OnInit - 初始化                                                  |
//+------------------------------------------------------------------+
int OnInit()
{
    // 设置缓冲区(数据序列)
    SetIndexBuffer(0, DeviationBuffer, INDICATOR_DATA);

    // 创建 MA 指标句柄
    g_maHandle = iMA(_Symbol, _Period, MaPeriod, 0, MODE_EMA, PRICE_CLOSE);
    if(g_maHandle == INVALID_HANDLE) return INIT_FAILED;

    return INIT_SUCCEEDED;
}

//+------------------------------------------------------------------+
//| OnCalculate - 每次有新 K 线或 Tick 调用                          |
//+------------------------------------------------------------------+
int OnCalculate(const int rates_total,
                const int prev_calculated,
                const datetime &time[],
                const double &open[],
                const double &high[],
                const double &low[],
                const double &close[],
                const long &tick_volume[],
                const long &volume[],
                const int &spread[])
{
    // 只在新增 K 线时计算
    if(rates_total <= MaPeriod) return 0;
    int start = prev_calculated == 0 ? MaPeriod : prev_calculated - 1;

    // 取 MA 数据
    double ma[];
    ArraySetAsSeries(ma, true);
    if(CopyBuffer(g_maHandle, 0, 0, rates_total, ma) <= 0) return 0;

    // 计算偏离百分比 = (close - ma) / ma × 100
    for(int i = start; i < rates_total; i++)
    {
        if(ma[i] != 0)
            DeviationBuffer[i] = (close[i] - ma[i]) / ma[i] * 100;
    }

    return rates_total;
}
```

### 3.3 使用方式
1. 复制到 `MT5\MQL5\Indicators\PriceDeviation.mq5`,按 F7 编译
2. 在 MT5 终端"导航" → "指标" → 找到 PriceDeviation → 拖到 EURUSD 图表
3. 副图会出现一条蓝色曲线,显示价格偏离 MA20 的百分比

## 四、插件示例 2: Script(一次性脚本)

**Script** 执行一次即结束,常用于批量操作工具。

### 4.1 业务场景
一键平掉当前品种的所有持仓(包括手动单和 EA 单)。

### 4.2 源码

```mql5
//+------------------------------------------------------------------+
//|                                          CloseAll.mq5            |
//|       华鑫融汇 - 脚本示例: 一键平仓                                |
//+------------------------------------------------------------------+
// 【目的】执行一次,立即平掉当前品种所有持仓
//        用法: 双击脚本 → 自动平仓 → 脚本结束
//+------------------------------------------------------------------+

#include <Trade\Trade.mqh>

CTrade g_trade;

//+------------------------------------------------------------------+
//| OnStart - 脚本入口(执行一次)                                     |
//+------------------------------------------------------------------+
void OnStart()
{
    int closedCount = 0;
    int failedCount = 0;

    // 倒序遍历(平仓后 PositionsTotal 会立刻变化)
    for(int i = PositionsTotal() - 1; i >= 0; i--)
    {
        ulong ticket = PositionGetTicket(i);

        // 只平当前品种
        if(PositionGetString(POSITION_SYMBOL) != _Symbol) continue;

        if(g_trade.PositionClose(ticket))
        {
            Print("【平仓成功】ticket=", ticket);
            closedCount++;
        }
        else
        {
            Print("【平仓失败】ticket=", ticket, " err=", g_trade.ResultRetcodeDescription());
            failedCount++;
        }
    }

    Print("=== 完成 ===");
    Print("  成功: ", closedCount);
    Print("  失败: ", failedCount);
}
```

### 4.3 使用方式
1. 复制到 `MT5\MQL5\Scripts\CloseAll.mq5`,按 F7 编译
2. 在 MT5 终端"导航" → "脚本" → 找到 CloseAll → 双击
3. 脚本执行一次后结束,查看"专家"标签页看结果

## 五、EA / Indicator / Script 三种程序对比

| 维度 | EA | Indicator | Script |
|------|----|-----------|--------|
| 触发方式 | 每个 Tick 触发 OnTick | 每个 Tick 触发 OnCalculate | 用户双击触发 OnStart |
| 生命周期 | 长期运行(随图表) | 长期运行(随图表) | 一次执行即结束 |
| 下单能力 | ✅ 可以 | ❌ 不可以 | ✅ 可以 |
| 画图能力 | ✅ ObjectCreate | ✅ 指标缓冲区 | ✅ ObjectCreate |
| 典型用途 | 自动化交易 | 辅助分析画图 | 批量平仓/数据导出 |
